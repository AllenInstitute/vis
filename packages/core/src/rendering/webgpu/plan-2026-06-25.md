# Plan: Unified WebGPU Rendering & Memory Architecture

Replace the current fragmented binding/render graph code with a layered, persistent, mutation-driven system that mirrors the user's 9-step workflow: `ResourceSlot` → `WgslShader` → rooted `BindingGraph` (group tree) → `Resource` (data-bearing, backed by `BufferManager`) → `Drawable` → persistent `Scene` (render graph) → state-deduplicating `Encoder`. Every device-scoped operation funnels through a `RenderingContext` instance (pipeline build, future buffer/drawable/encoder construction). Keep `resources/` (after rename), `shaders/`, `memory/`. Replace `graphs/nodes.ts` and `pipelines/binding-graphs.ts` wholesale. Adopt `webgpu-utils` as a dependency to eliminate hand-written WGSL introspection / structured buffer view / BGL descriptor / vertex-buffer interleave code.

## Locked decisions
- Existing `Resource` → `ResourceSlot`. New data-bearing object inherits the name `Resource`.
- `BindingGraph` is a **rooted structure**: each `bindings()` call produces a graph whose nodes form a single tree rooted at one `BindingGroup`. One `ResourceSlot` may appear in multiple unrelated graphs (slot ownership is graph-local), but never twice within one graph.
- **Binding-graph authoring is variadic-children, parent-down**: `group(spec?, ...children)` where each child is either a `ResourceSlot` or a nested `BindingGroup`. The optional first arg is a `GroupSpec` (`label`, `maxDepth`); subsequent args are bucketed at runtime via `isResourceSlot` / `isBindingGroup`. **`bindings()` is variadic over shaders**: `bindings(root: BindingGroup, ...shaders: WgslShader[])`.
- **`RenderingContext` is the device-scoped state owner.** Free `pipeline()` is removed; the class-only API is `ctx.pipeline(graph, shader, state)`. Forward-looking: `ctx.resource(slot, init?)` (Phase 4), `ctx.drawable({...})` (Phase 5), `ctx.encoder()` and `ctx.submit(scene)` (Phase 7) all funnel through the class. The class owns the pipeline cache (instance-scoped, replacing the old `WeakMap<GPUDevice, Map<...>>` module-level cache) and, per phase, the bind-group cache.
- **`BufferManager` is always externally constructed.** The context holds a reference (`ctx.bufferManager`) but never owns the lifetime. `ctx.dispose()` clears the pipeline cache only — it never cascades into `bufferManager.dispose()`. There is no `memory: {maxBytes, ...}` auto-construct shortcut on `RenderingContextSpec`, in Phase 3 or any later phase.
- `Scene` (the persistent render graph) is **mutable**: `add`/`remove`/`replace`/`markDirty`; Encoder caches per-subtree commands.
- `graphs/nodes.ts` + `pipelines/binding-graphs.ts` replaced wholesale.
- **Resource ownership**: strict refcount with `share()` for explicit cross-Drawable sharing.
- **State nodes**: discriminated variants (`ViewportNode`, `ScissorNode`, …) — not a single `StateNode{kind,value}`.
- **No `PipelineRefNode` in the Scene.** Drawable owns its pipeline assignment (singular). Multi-pipeline drawing = multiple Drawable instances sharing geometry via `Resource.share()`. State nodes sit *above* drawables; Encoder emits `setPipeline` lazily when a Drawable's pipeline differs from `ActiveState.pipeline`.
- **Worker-friendly**: all public constructors take plain-object descriptors (no closures). Scene mutations serialize as messages. Encoder/Pipeline/Resource instances are thread-local but reconstructable from descriptors on either side.
- **Public API style**: single-word, lowercase functions wherever feasible. Compound-word factories grouped under namespaces (e.g., `slot.uniform`, `slot.storageTexture`). Device-scoped stateful operations live on the `RenderingContext` class (class export + lowercase `renderingContext()` factory).
- **`webgpu-utils` dependency** for: `makeShaderDataDefinitions` (WGSL introspection), `makeStructuredView` (typed CPU views over buffers), `createBuffersAndAttributesFromArrays` (interleaved vertex buffer **layouts only** — never used for GPU buffer creation), `createTextureFromSource`/`Image`/`Images`, `numMipLevels`, `generateMipmap`. (Bind-group layouts are constructed manually inside `buildPipeline` from `shaderSlotEntries` + `toBindGroupLayoutEntry`; `makeBindGroupLayoutDescriptors` is not used.)
- **`override` semantics**: slot-by-slot. `BindingOverrideNode.overrides: Map<ResourceSlot, Resource>` replaces specific slots for its subtree; non-overridden slots fall through to the Drawable's own bindings.
- **State restoration**: state nodes (`viewport`, `scissor`, `stencilref`, `blendconstant`) **restore-on-exit** via `ActiveState` snapshots captured at subtree entry. Mental model: nested lexical scope.
- **Pipeline state lives at build time, not in the binding graph**. The binding graph's groups carry only resource layout; rendering state (primitive / depthStencil / multisample / blend / fragment targets) is supplied at `ctx.pipeline(graph, shader, state)` call time. The same `shader` can be built any number of times against one context with different states — each produces a distinct `BuiltPipeline`. `BuiltPipeline.fingerprint` incorporates `(shader.id, slotIndex, normalizedState)` and keys the per-context cache.
- **Slots carry their struct type via the shader DSL**, not raw WGSL strings. `slot.uniform({ name, type: structDecl })` where `structDecl = struct('Camera', [member('view', 'mat4x4f'), ...])`. Slot construction emits the struct to WGSL once internally and feeds it to `webgpu-utils.makeShaderDataDefinitions` so the slot self-describes its layout — `resource(slot)` works without needing a `BuiltPipeline.defs` round-trip. Struct descriptors are reusable across slots and shaders.
- **`drawable.reuse(other, overrides)`** convenience method auto-shares vertex/index Resources and inherits unspecified bindings from `other`; explicit `share()` increments refcounts.
- **BufferManager owns all GPU buffer creation**. `device.createBuffer` is forbidden anywhere outside `BufferManager` implementations. Every caller obtains a buffer via `BufferManager.acquire(size, usage)` and receives a `BufferHandle { gpu: GPUBuffer, offset: number, size: number, … }`. This makes slab-style managers (one giant `GPUBuffer` carved into slices) a drop-in replacement for `BatchPoolBufferManager` without touching callers. All WebGPU calls thread `handle.offset` + `handle.size`: `setVertexBuffer(slot, gpu, offset, size)`, `setIndexBuffer(gpu, format, offset, size)`, bind-group entries `{ buffer: gpu, offset, size }`, `queue.writeBuffer(gpu, offset, data)`. Alignment (uniform/storage `minOffsetAlignment`, vertex/index 4-byte) is the manager's responsibility.
- **End-to-end type safety, Tier 1 only in v1**: `struct → slot → resource → resource.set({...})` is fully typed. `member()` / `struct()` use `<const>` generics; the `StructDecl<TsShape>` carries forward through `slot.uniform<T>`, `ResourceSlot<Kind, T>`, `resource(slot): BufferResource<T>`, and `BufferResource.set(values: Partial<T>)` + `setField<K extends keyof T>(k: K, v: T[K])`. Tier 2 (drawable bindings type-checked against pipeline slot map) and Tier 3 (shader-declarations → pipeline slot inference) are deferred — the v1 API surface is designed to be generic-ready (internal `unknown`s where Tiers 2/3 would inject type params) so they can be added post-v1 without breaking changes.

## Phases

### Phase 1 — Rename `Resource` → `ResourceSlot` + freeze public API surface + struct-DSL slots
*Prep; runs first to free the `Resource` name.*
- Rename in `resources/resource.ts`: type, brand symbol (→ `vis-core.webgpu.ResourceSlot`), internal constructors.
- Update `bind.ts`, `bound.ts`, `pipelines/resources.ts`, `pipelines/draw-context.ts`, `pipelines/bind-group-cache.ts`, `pipelines/traversal.ts`, examples/tests.
- `BindingMap` becomes `ReadonlyMap<ResourceSlot, {group, binding}>`.
- **Slot factories take struct descriptors from the shader DSL** (not raw WGSL):
  - `slot.uniform({ name, type: StructDecl }): ResourceSlot` — internally renders `type` to WGSL and runs `makeShaderDataDefinitions` once at slot construction; the slot caches its `ShaderDataDefinitions` entry so `resource(slot)` is self-sufficient.
  - `slot.storage({ name, type: StructDecl, access: 'read' | 'read_write' })`.
  - `slot.texture({ name, sampleType, viewDimension? })`, `slot.storageTexture({ name, format, viewDimension? })`, `slot.sampler({ name, type? })`, `slot.external({ name })`.
  - **Const-generic API surface from day one**: `member<const Name extends string, const WgslType extends WgslPrimitive>(name, type)` and `struct<const Members extends readonly Member[]>(name, members)` produce a `StructDecl<TsShape>` where `TsShape` is derived from the members tuple via a `WgslToTs<T>` conditional-type map (mat4x4f → `Float32Array | readonly number[]`, vec3f → `[number, number, number] | Float32Array`, scalars → `number`, etc.). `slot.uniform`/`slot.storage` carry the parameter forward: `slot.uniform<T>({ name, type: StructDecl<T> }): ResourceSlot<UniformKind, T>`. Tier 1 lights up in Phase 4 the moment `BufferResource<T>.set(values: Partial<T>)` is implemented — no breaking API change. Tier 2/3 generics (drawable bindings type-checked against pipeline; shader-declarations inferred into pipeline slot map) remain `unknown` placeholders in v1 so they can be added later without breaking changes.
- `WgslShader` constructor assigns each instance an internal `id: string` (UUID-ish) — used by `bindings()` to index leaves and by `pipeline()`/encoder fingerprinting.
- Expose public-API namespace from `packages/core/src/rendering/webgpu/index.ts`:
  - `slot.{uniform,texture,sampler,storage,storageTexture,external}` factory namespace
  - `shader`, `struct`, `member` re-exports from `shaders/`
  - Reserve type stubs (interfaces / `unknown`-returning declarations) for `group`, `bindings`, `renderingContext`, `RenderingContext`, `resource`, `drawable`, `scene`, `viewport`, `scissor`, `stencilref`, `blendconstant`, `container`, `override`, `draw` so downstream code can be type-checked against the final shape from day one. (`pipeline`, `encoder`, `submit` are NOT in the stub list — they become instance methods on `RenderingContext` rather than free functions.)
- Add `webgpu-utils` to `packages/core/package.json` dependencies.

### Phase 2 — Tree `BindingGraph` + variadic `group()` and `bindings()`
*Depends on Phase 1.*
- Replace `pipelines/binding-graphs.ts` with `pipelines/binding-graph.ts`:
  - `BindingGroup { id, label?, slots: readonly ResourceSlot[], subgroups: readonly BindingGroup[], maxDepth: number }`. Frozen. **No `parent`, no `depth`** — those are graph-local, not group-local (the same group may sit at different depths inside different graphs).
  - `BindingGraph { root, shaders, groups, _slotIndex: ReadonlyMap<ResourceSlot, {group, binding}>, _groupDepth: ReadonlyMap<BindingGroup, number> }`. Frozen.
  - **Shaders are NOT graph members.** Shaders are consumers passed into `bindings(root, ...shaders)` and validated against the tree; the graph stores them as metadata so `ctx.pipeline(graph, shader, state)` can sanity-check at build time.
- `group(spec?, ...children)`: if the first arg is a `GroupSpec` (detected via brand absence on `isBindingGroup` / `isResourceSlot`), it carries `label` / `maxDepth`; subsequent args are bucketed at runtime into `slots` vs `subgroups`. Returns a frozen `BindingGroup`.
- `bindings(root: BindingGroup, ...shaders: readonly WgslShader[]): BindingGraph` — iterative DFS over the tree; builds `_slotIndex` (depth-from-root → `group`; order-within-group → `binding`) and `_groupDepth`; enforces (a) per-node `maxDepth`, (b) within-tree slot uniqueness, (c) every declared shader slot is present in the tree. Returns a frozen graph. Throws when no shaders are supplied.
- One `ResourceSlot` may appear in **multiple unrelated graphs** (slot ownership is graph-local). It may NOT appear twice within one graph.
- New `traverse.ts`:
  - `resolveShaderBindings(graph, shader): Map<ResourceSlot, {group, binding}>` — reads `graph._slotIndex` filtered to the shader's declared slots.
  - `shaderSlotEntries(graph, shader): Array<{slot, group, binding}>` — same data, array form used by build-time BGL construction.

### Phase 3 — Pipeline build + per-context cache: `RenderingContext`
*Depends on Phase 2.*
- New `webgpu/context.ts`:
  - `class RenderingContext` + lowercase factory `renderingContext(spec)`. `RenderingContextSpec = { device: GPUDevice; label?: string; bufferManager?: BufferManager }`. The `bufferManager` slot is accepted but unused by Phase 3 — Phase 4 wires `ctx.resource()` over it. **`BufferManager` is always externally constructed**; the context holds a reference and never disposes it.
  - Owns the pipeline cache as `Map<fingerprint, BuiltPipeline>` (instance-scoped). Two contexts against the same `GPUDevice` get independent caches.
  - `ctx.pipeline(graph, shader, state): BuiltPipeline` — `normalizePipelineState` → `resolveShaderBindings` → `pipelineFingerprint` → cache check → `buildPipeline` → cache set. Asserts `!disposed` and includes the context label in the use-after-dispose error.
  - `ctx.disposePipelineCache()` clears the map but does NOT mark the context disposed. `ctx.dispose()` calls `disposePipelineCache()` + flips `_disposed`; idempotent. **Does NOT cascade into `bufferManager.dispose()`.**
  - `ctx.stats(): { pipelines: number }` — extended per phase (Phase 4 adds memory fields; Phase 7 adds bind-group fields).
- New `pipelines/build.ts` — internal `buildPipeline(device, graph, shader, normalizedState, slotIndex, fingerprint): BuiltPipeline`. Pure function; not exported from the public barrel:
  - `bindShader(shader, slotIndex)` → `asSource(...)` → `makeShaderDataDefinitions(wgsl)` → manual BGL construction (bucket `shaderSlotEntries` by depth → `toBindGroupLayoutEntry` → `device.createBindGroupLayout` per group) → `device.createPipelineLayout` → `device.createShaderModule` → compose `GPURenderPipelineDescriptor` from `normalizedState` → `device.createRenderPipeline`. Returns a frozen `BuiltPipeline`.
  - `BuiltPipeline { id, gpu, layout, bindGroupLayouts, slotIndex, defs, fingerprint, state, shader }`. `fingerprint` keys the per-context cache and drives encoder no-op elision. `state` and `shader` are stashed for downstream encoder / debug use.
- New `pipelines/fingerprint.ts` — djb2-32 hash of `(shader.id, slotIndex, normalizedState)` → `pl_<hex>`.
- New `pipelines/pipeline-state.ts` — Zod schema + `normalizePipelineState` (canonical, key-order-independent, defaults `vs_main` / `fs_main`).
- Content-addressable: same `(graph, shader, state)` against the same context ⇒ same fingerprint ⇒ same `BuiltPipeline`. Different contexts ⇒ same fingerprint but distinct GPU pipelines.

### Phase 4 — Data-bearing `Resource` + BufferManager extensions: `ctx.resource()`
*Depends on Phase 1 for freed name; structurally independent of Phases 2/3 except that `ctx.resource()` lives on the Phase-3 `RenderingContext`.*
- New `webgpu/data/`:
  - `resource.ts`: `Resource` discriminated union — `BufferResource<T>`, `TextureResource`, `SamplerResource`, `StorageBufferResource<T>`, `StorageTextureResource`, `ExternalTextureResource`. `<T>` is the Tier-1 type parameter carried from the slot's `StructDecl<T>`.
  - **`ctx.resource(slot, init?): Resource`** — instance method on `RenderingContext` (replaces the originally-planned free function). Dispatches on `slot.kind`. `init` is kind-specific (initial data for buffers; image/canvas/data for textures via `webgpu-utils.createTextureFromSource`).
    - Buffer-backed kinds (`uniform` / `storage`) consult `ctx.bufferManager`. If `ctx.bufferManager === undefined`, throws a clear error: `RenderingContext '<label>': resource(slot 'foo') requires a bufferManager; pass one to renderingContext({ device, bufferManager })`.
    - Sampler / texture / external-texture kinds don't need the BM and work on a context without one.
  - `BufferResource<T>`:
    - Holds a `BufferHandle` from `BufferManager`.
    - Uses `webgpu-utils.makeStructuredView(slot.def, optionalArrayBuffer)` for typed views. **Size derives from the slot's own cached `ShaderDataDefinitions` entry (populated at slot construction in Phase 1) — no dependency on a `BuiltPipeline` is required.** A `BuiltPipeline.defs` round-trip is *only* used when the slot kind doesn't carry its own struct (e.g., unsized storage arrays whose size is configured per-binding).
    - `set(values: Partial<T>)` / `setField<K extends keyof T>(k: K, v: T[K])` — **fully typed against the slot's struct** (Tier 1).
    - `commit(device)` flushes the CPU `arrayBuffer` to GPU via `queue.writeBuffer(handle.gpu, handle.offset, arrayBuffer)`. Bumps `version` for bind-group cache invalidation. **Always passes `handle.offset`** so slab-style managers work transparently.
    - `usage` declared at construction (defaulted from the slot kind); validated against slot at bind time.
    - `destroy()` releases `BufferHandle`. `share()` increments refcount, returns `this`.
  - `TextureResource` factory delegates to `webgpu-utils.createTextureFromSource`/`Image`/`Images`; mip handling via `numMipLevels` + `generateMipmap`. (Textures are NOT slab-managed; `TextureResource` directly owns its `GPUTexture` in v1. Future managers may wrap textures, but `BufferManager` stays buffer-only.)
- Extend `memory/`:
  - **`BufferHandle` carries `{ gpu: GPUBuffer, offset: number, size: number, … }`** as required fields. `BatchPoolBufferManager` uses `offset: 0, size: bucketSize`. A future `SlabBufferManager` carves non-zero offsets out of one giant buffer with no change to consumers.
  - `acquireForSlot(slot: ResourceSlot, sizeBytes, usage): BufferHandle` on `BufferManager` — validates `usage & required === required` and applies the slot kind's required alignment (uniform/storage `minOffsetAlignment` from `device.limits`).
  - `precheck(budgetCost: number): boolean` — non-allocating budget check used by `ctx.resource()` and `ctx.drawable()` to fail fast before partial allocation.
  - `OutOfBudgetError`/`OutOfBucketError` surface at the construction site.
  - **Hard rule (documented + lint-checked)**: `device.createBuffer` may only be called from inside a `BufferManager` implementation. Any other call site is a code-review-blocking error.
- **Ownership reminder**: `RenderingContext` never creates, manages, or disposes the `BufferManager` — the caller does. `ctx.dispose()` clears the pipeline cache but leaves the BM untouched. Extend `ctx.stats()` here with `{ bytes, leasedBytes }` pulled from `bufferManager.stats()` (read-through; the context does not duplicate the data).

### Phase 5 — `Drawable`: `ctx.drawable()`
*Depends on Phases 3 + 4.*
- New `webgpu/drawable.ts`:
  - `Drawable { id, pipeline: BuiltPipeline, vertexBuffers: Map<location, BufferResource>, indexBuffer?: { resource, format }, bindings: Map<ResourceSlot, Resource>, draw: DrawCall }`.
  - `DrawCall = { kind:'array', vertexCount, instanceCount, firstVertex, firstInstance } | { kind:'indexed', indexCount, instanceCount, firstIndex, baseVertex, firstInstance }`.
  - **`ctx.drawable({ pipeline, vertex, index?, bindings, draw }): Drawable`** — instance method on `RenderingContext` (replaces the originally-planned free function):
    - `vertex` may be a `Map<location, BufferResource>` (assumes pre-built) or a raw arrays object. In the raw-arrays case: `webgpu-utils.createBuffersAndAttributesFromArrays` is used **only to derive the interleaved byte layout, attribute formats, and `bufferLayouts`**. **GPU buffer allocation goes through `ctx.bufferManager.acquire(sizeBytes, GPUBufferUsage.VERTEX | COPY_DST)`** (and `GPUBufferUsage.INDEX | COPY_DST` for the index buffer); upload via `ctx.device.queue.writeBuffer(handle.gpu, handle.offset, interleavedBytes)`; wrap each in a `BufferResource`. **`device.createBuffer` is never called from this code path** — the buffer manager owns all GPU buffer creation so slab-style managers remain transparent.
    - Raw-arrays path requires `ctx.bufferManager`; pre-built `BufferResource` path does not. Same throw-without-BM rule as `ctx.resource()`.
    - `index` similarly: a `BufferResource` + format, or raw indices that go through `ctx.bufferManager` for the GPU buffer.
    - `bindings` is a `Record<slotName, Resource>` validated against `pipeline.slotIndex`: every required slot supplied; each `Resource.usage` compatible with its slot.
  - Multi-pipeline pattern: construct N Drawables sharing the same vertex/index/binding Resources via `Resource.share()`. A convenience helper auto-shares vertex/index resources and validates a new pipeline — placement TBD during implementation (method on `Drawable` vs. method on `RenderingContext`; see Open considerations).
  - `destroy()` decrefs every owned `Resource` (vertex BufferResources, index BufferResource, and any owned binding Resources — `share()`'d ones are not destroyed until their last owner releases).

### Phase 6 — Persistent `Scene` + state-node factories
*Depends on Phase 5.*
- New `webgpu/scene/`:
  - `types.ts`:
    - `Scene { id, target, root: SceneNode, parents: Map<NodeId, NodeId>, dirty: Set<NodeId> }`
    - `RenderTarget = { color: GPURenderPassColorAttachment[], depthStencil?: GPURenderPassDepthStencilAttachment }`
    - `SceneNode` discriminated union (**no `PipelineRefNode`**):
      - `ContainerNode { id, kind:'container', children }`
      - `ViewportNode { id, kind:'viewport', x, y, width, height, minDepth, maxDepth, children }`
      - `ScissorNode { id, kind:'scissor', x, y, width, height, children }`
      - `StencilRefNode { id, kind:'stencilref', value, children }`
      - `BlendConstantNode { id, kind:'blendconstant', color, children }`
      - `BindingOverrideNode { id, kind:'override', overrides: Map<ResourceSlot, Resource>, children }`
      - `DrawableNode { id, kind:'draw', drawable: Drawable }` — leaf.
  - `scene.ts`:
    - `scene(descriptor)` constructs from a worker-transferable POJO.
    - Public factories (each single-word lowercase): `container(children)`, `viewport({...})`, `scissor({...})`, `stencilref(value, children)`, `blendconstant(color, children)`, `override(map, children)`, `draw(drawable)`.
    - `Scene` methods: `add(parentId, node) → NodeId`, `remove(id)`, `replace(id, node)`, `markDirty(id)`, `markSubtreeDirty(id)`. O(depth) dirty propagation via `parents`.
    - `'structure-changed'` event fired after each mutation for downstream cache trimming.
- **Worker shape**: every node descriptor is a POJO referencing only string IDs (pipeline/resource/drawable IDs). The scene itself is `structuredClone`-able. The thread that owns the encoder keeps an `id → live-object` dictionary; mutations from the other thread arrive as messages and replay via the same public methods.
- **Compute-future-proofing**: the base `Graph<NodeKind>` interface (which `Scene` extends) is parameterized; a future `ComputeScene`/`ComputeGraph` reuses `parents`/`dirty`/event infrastructure with different node variants. Not implemented in v1.

### Phase 7 — `Encoder`: `ctx.encoder()` + `ctx.submit(scene)`
*Depends on Phases 2, 3, 6. Fulfills the existing `plan-2026-06-01.md`.*
- New `webgpu/encoder/`:
  - `state.ts`: `ActiveState { pipeline?, bindGroups: (GPUBindGroup|undefined)[], vertexBuffers: Map<slot,BufferHandle>, indexBuffer?, viewport?, scissor?, stencilRef?, blendConstant? }`. Fingerprint-able for snapshot comparison. Vertex/index entries store the full handle (gpu + offset + size) so equality checks correctly distinguish two slices of the same underlying slab buffer.
  - `bind-group-builder.ts`: takes a Pipeline + scene-resolved Resources + any `BindingOverrideNode` overrides + Drawable bindings → `GPUBindGroup[]` keyed by `(pipelineId, groupIndex, ...resourceVersions)`. The bind-group cache now lives on `RenderingContext` (the builder consults `ctx`, not module-level state); cache entries are keyed by the same tuple and invalidated by `Resource.version` bumps. **Bind-group entries always specify `offset` + `size` from the `BufferHandle`** — never assume offset 0 or whole-buffer size, so slab managers work transparently.
  - `encoder.ts`: `GraphEncoder` per `plan-2026-06-01.md`:
    - Two-phase per frame: **plan** (walk dirty subtrees, recompute command lists, restore snapshots) → **execute** (replay cached + freshly-encoded segments in correct child order against a `GPURenderPassEncoder`).
    - State elision via `ActiveState` comparison; per-subtree command cache invalidated by `dirty`.
    - Entry/exit `ActiveState` snapshots so a clean cache subtree can be replayed without re-walking.
    - **State restore-on-exit**: on entering a state-node subtree (`ViewportNode`, `ScissorNode`, `StencilRefNode`, `BlendConstantNode`), the encoder snapshots the relevant field of `ActiveState` and emits the new setter; on exit, if the snapshotted value differs from the value left active, emits the restore setter. Bind-group overrides via `BindingOverrideNode` snapshot/restore the affected bind groups identically.
    - **Pipeline emission**: on visiting a `DrawableNode`, if `drawable.pipeline.id !== activeState.pipeline?.id`, emit `setPipeline` and invalidate incompatible bind-group slots (per WebGPU rules, with `slotIndex` layout compatibility). Then `setBindGroup` for slots that changed, then `setVertexBuffer(loc, handle.gpu, handle.offset, handle.size)` and `setIndexBuffer(handle.gpu, format, handle.offset, handle.size)`, then `draw`/`drawIndexed`. Pipeline state is **not** auto-restored when leaving a subtree — pipeline is a draw-call attribute, not a scoped state node.
  - **`ctx.encoder(): GraphEncoder`** — instance method on `RenderingContext` (replaces the originally-planned free function); the encoder is bound to `ctx.device` and consults `ctx`'s bind-group cache.
  - **`ctx.submit(scene): void`** — instance method that constructs (or reuses) an encoder, runs plan + execute, and submits the `GPUCommandBuffer` to the device queue. `encoder.submit(scene)` is available symmetrically; final placement TBD during implementation (see Open considerations).
  - `ctx.stats()` extends with `{ bindGroups: number, ... }` to surface bind-group cache occupancy.

### Phase 8 — Memory lifecycle & safeguards
*Crosscuts Phases 4–7.*
- `Resource.destroy()` → `BufferHandle.release()` (or texture/sampler destroy).
- `Drawable.destroy()` → decref owned `Resource`s.
- `Scene.remove(id)` → decref Drawables in removed subtree (caller policy whether decref ⇒ destroy).
- Encoder holds weak references to `GPUBindGroup`s in cache; on `Resource.destroy()`, sweep cache entries whose version vector references the destroyed resource.
- Budget surfacing: `BufferManager.precheck()` used by `ctx.resource()` (Phase 4) and `ctx.drawable()` (Phase 5).
- `Scene.on('structure-changed', ...)` is the integration point for trimming bind-group cache after subtree removals.
- **`RenderingContext.dispose()` clears the pipeline cache (Phase 3) and the bind-group cache (Phase 7).** It does NOT dispose the externally-supplied `bufferManager` — that lifetime is owned by the caller, who must call `bufferManager.dispose()` separately when done.

## Scope boundaries
- **In**: render pipelines, persistent `Scene` (render-graph), Encoder, Drawables, `BufferResource<T>` via `BufferManager`-owned buffer allocation with slab-ready `BufferHandle`, Tier-1 typed `resource.set`, `webgpu-utils` integration, Worker-compatible API shape (verified by example).
- **Deferred**: compute pipelines / ComputeScene (design Phase 6 base types so they fit later), texture/sampler pooling (treat as caller-owned for now; `TextureResource` is a thin wrapper), multi-render-pass scheduling within one scene (one scene = one render pass for v1), integration with legacy WebGL `RenderServer` / `beginLongRunningFrame`, **Tier-2 type-safety** (drawable bindings checked against pipeline slot map at compile time), **Tier-3 type-safety** (shader declarations → pipeline slot inference), **SlabBufferManager implementation** (mock used in tests; full impl is a follow-up since the architecture supports it without API change).
- **Removed**: `graphs/nodes.ts` (`mergeGraphs`, `mergeCommandAndBindGroupGraphs`), old chain-style `BindingGraph`.
